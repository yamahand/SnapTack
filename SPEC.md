# SnapTack 仕様書 (v1.0)

## 1. 概要

SnapTack は、画面の任意の範囲をキャプチャし、その画像を「付箋(スクラップ)」として画面上に貼り付けておける Windows 用常駐ツールである。SETUNA2(開発終了したフリーソフト)の後継を目指す。

コンセプトは **「情報の一時保持」**。資料の一部・エラーメッセージ・参考画像などを一瞬で切り取って画面に貼っておき、見比べたり、他アプリへコピーしたりできる。

- 公開先: GitHub (ソース・リリース) / itch.io
- ライセンス: MIT License (LICENSE)
- 開発形態: 個人開発

## 4. 機能仕様 (v1.0)

### 4.1 起動・常駐

- 起動時にウィンドウは表示せず、タスクトレイに常駐する。
- 二重起動は禁止 (Mutex)。二重起動時は何も表示せず即終了する。
- トレイアイコンのメニュー:
  - キャプチャ (ホットキー表記付き)
  - 設定
  - 終了
- トレイアイコンのダブルクリックでもキャプチャを開始する。

### 4.2 キャプチャ開始 (ホットキー)

- デフォルトホットキー: `Ctrl+Shift+Z`
- ホットキー登録に失敗した場合(他アプリと競合)は、警告を通知しトレイメニューからの操作は引き続き可能とする。
- キャプチャの流れ:
  1. ホットキー押下の瞬間に **画面全体を1枚キャプチャ(フリーズ)** する。
  2. そのフリーズ画像を全画面オーバーレイとして表示する(この方式によりオーバーレイ自身の映り込み・チラつきがなく、動画等の一瞬も切り取れる)。

### 4.3 範囲選択オーバーレイ

- 全画面に薄暗いレイヤを重ね、ドラッグ中の選択範囲だけ明るく(くり抜き)表示する。
- 選択中は選択範囲のサイズ (幅 × 高さ、物理ピクセル) をラベル表示する。
- 操作:
  - 左ドラッグ: 範囲選択。マウスを離すと確定。
  - `Esc` または右クリック: キャンセル。
  - 数 px 未満の極小ドラッグ(誤クリック)はキャンセル扱い。
- オーバーレイ表示中の再キャプチャ要求(ホットキー再押下)は無視する。

### 4.4 付箋 (スクラップ)

- 選択確定と同時に、選択範囲の画像が **元の画面位置にピッタリ重なる形** で付箋ウィンドウとして生成される(「画面から剥がした」体験)。
- 見た目:
  - 枠線 1px (グレー系)
  - 影なし
  - 常に最前面 (Topmost)
  - タスクバーに表示しない
- 操作:
  | 操作 | 動作 |
  |---|---|
  | 左ドラッグ | 移動 |
  | `Ctrl+C` (フォーカス中) | 画像をクリップボードへコピー |
  | 中クリック | 閉じる |
  | 右クリック | コンテキストメニュー表示 |
- コンテキストメニュー: 「コピー (Ctrl+C)」「閉じる (中クリック)」の2項目+セパレータのみ。
- 付箋は複数枚同時に存在できる。全部閉じてもアプリは終了しない。
- 付箋は画面キャプチャと同じ物理解像度で等倍表示する(拡大縮小しない)。

### 4.5 設定画面

- v1.0 の設定項目はホットキーの変更のみ。
- トレイメニューの「設定」から開く。Fluent テーマ (`ThemeMode="System"`) を適用する。
- 設定はポータブル運用を考慮し、exe と同階層(書き込み不可の場合は `%APPDATA%\SnapTack`)に JSON で保存する。

### 4.6 終了

- トレイメニューの「終了」でアプリを終了する。開いている付箋はすべて閉じる。

## 5. 非機能要件

- **DPI**: Per-Monitor V2。キャプチャ・切り出しは物理ピクセルで行い、ウィンドウ配置は DIP に変換する。100%/125%/150%/175% スケーリングで付箋の位置ズレがないこと。
- **起動速度**: 常駐開始まで体感即時(1秒未満目安)。
- **配布形態**:
  - ポータブル版: zip 展開のみで動作 (自己完結型 single-file publish)
  - インストーラー版: 形式は M5 で選定 (Inno Setup 想定)
- **多言語**: v1.0 は日本語のみ。将来の英語化を見据え、UI 文字列は可能な範囲で一元管理する。

## 6. スコープ外 (将来バージョン)

| バージョン | 機能 |
|---|---|
| v1.x | 付箋の最小化(サイコロ化)、不透明度変更、PNG 保存、マルチディスプレイ対応 |
| v2.0+ | スクラップリスト、ゴミ箱(閉じた付箋の復元)、スタイル機能 |

v1.0 ではこれらを実装しないが、妨げない設計とする(例: キャプチャの抽象化、付箋の状態をモデルとして分離)。
