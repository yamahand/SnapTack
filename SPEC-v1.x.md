# SnapTack 仕様書 追補 (v1.x)

本書は SPEC.md (v1.0) への追補である。v1.0 の仕様は特記なき限り維持する。
リポジトリでは本書の内容を SPEC.md に統合してもよいし、追補として並置してもよい。

## 1. v1.x の機能一覧とリリース区切り

| リリース | 機能 |
|---|---|
| v1.1 | PNG 保存、不透明度変更 |
| v1.2 | 最小化(サイコロ化) |
| v1.3 | マルチディスプレイ対応 |

各リリースは独立して出荷可能とする。

## 2. 機能仕様

### 2.1 PNG 保存 (v1.1)

- 付箋のコンテキストメニューに「PNGで保存...」を追加する。ショートカットは `Ctrl+S`。
- `SaveFileDialog` を表示する。
  - デフォルトファイル名: `SnapTack_yyyyMMdd_HHmmss.png`
  - デフォルト保存先: 前回保存したフォルダ(初回は「ピクチャ」)。設定ファイルに記憶する。
- 画像はキャプチャ時の物理ピクセル等倍で保存する(不透明度・枠線は反映しない。元画像そのまま)。
- 保存失敗時はメッセージボックスで通知し、付箋は維持する。

### 2.2 不透明度変更 (v1.1)

- 付箋上でマウスホイール回転により不透明度を変更する。
  - 下回転で下げる、上回転で上げる。ステップ 10%。
  - 現在値が 10% の倍数でない場合(プリセット 75% / 25% 適用後)、最初の 1 ステップは ±5% として 10% の倍数へスナップし、以降は ±10% とする(例: 75% → 上回転 80% / 下回転 70%)。
  - 範囲: 20%〜100%(誤操作で見失わないよう下限 20%)。
- コンテキストメニューにプリセット「100% / 75% / 50% / 25%」を追加する(現在値にチェック表示)。
- 不透明度はウィンドウ全体 (`Window.Opacity`) に適用する。
- 新規付箋は常に 100% で生成する(既定値の設定化は行わない)。
- コピー・PNG 保存される画像は不透明度の影響を受けない。

### 2.3 最小化: サイコロ化 (v1.2)

- 付箋をダブルクリックすると「サイコロ」(小さなタイル)に切り替わる。もう一度ダブルクリックで元に戻る。
- サイコロの仕様:
  - サイズ: 48×48 DIP 固定
  - 表示: 元画像を縮小したサムネイル(アスペクト比維持・中央クロップ)+ 枠線 1px
  - 位置: 切り替え時のウィンドウ左上を維持する。復元時も現在の左上位置を基準に展開するが、付箋が画面外へはみ出す場合は表示中のモニタ内に収まるよう位置をクランプする
  - サイコロ状態でもドラッグ移動・中クリック閉じる・右クリックメニューは有効
  - サイコロ状態の `Ctrl+C` / `Ctrl+S` は元画像に対して機能する
- コンテキストメニューに「サイコロ化 / 元に戻す」(状態に応じて表記切替、`ダブルクリック` 表記付き)を追加する。
- 不透明度はサイコロ化しても維持する。

### 2.4 マルチディスプレイ対応 (v1.3)

- キャプチャ開始時、**全モニタ**を対象とする。
- 実装方式: **モニタごとに個別のオーバーレイウィンドウ**を生成する。
  - モニタごとに個別に画面をフリーズキャプチャし、そのモニタのオーバーレイに表示する。
  - 理由: モニタごとに DPI が異なる環境 (Per-Monitor V2) で座標変換を正確に保つため。
- 範囲選択は **1つのモニタ内で完結** する(モニタをまたぐ選択は v1.x では非対応。ドラッグ中にカーソルが他モニタへ出た場合、選択中モニタの端でクリップする)。
- いずれかのモニタで選択が確定 / キャンセルされたら、全モニタのオーバーレイを閉じる。
- 付箋はキャプチャ元モニタの元位置に生成される。付箋自体は任意のモニタへドラッグ移動できる。
- 付箋の等倍表示は「キャプチャ元モニタの DPI」を基準とする。異なる DPI のモニタへ移動した際の挙動は設定で切り替えられるようにする:
  - **WPF 既定(規定値)**: WPF の Per-Monitor V2 既定動作に任せる。移動先モニタの DPI に合わせて再スケーリングされ、DIP サイズは維持されるが物理ピクセル等倍は崩れる。
  - **物理ピクセル等倍を維持**: `WM_DPICHANGED` を独自処理し、どのモニタ上でも物理ピクセルサイズを維持する。
- 上記の切り替え項目を設定画面に追加する。
- `ScreenCapturer` 抽象化は「全モニタ列挙」+「指定モニタのキャプチャ」を提供する形に拡張する。

## 3. 設定ファイルへの追加項目

| キー | 内容 | 追加リリース |
|---|---|---|
| `LastSaveDirectory` | PNG 保存の前回フォルダ | v1.1 |
| `KeepPhysicalPixelSize` | 異なる DPI のモニタへ移動した際に物理ピクセル等倍を維持するか(既定: `false` = WPF 既定動作) | v1.3 |

不透明度・サイコロ状態は永続化しない(付箋はセッション限りの存在のため)。

## 4. 互換性・非機能

- v1.0 の操作体系(左ドラッグ移動 / Ctrl+C / 中クリック / 右クリックメニュー)は変更しない。追加操作はすべて既存操作と競合しないこと。
- ダブルクリック(サイコロ化)とドラッグ移動の判定が干渉しないこと(ダブルクリック判定は移動を伴わないクリックに限る)。
- 混在 DPI 環境 (例: 150% + 100% の2画面) で、各モニタのキャプチャ・付箋位置がズレないこと。
