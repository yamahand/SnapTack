# デモ GIF の作り方

X (Twitter) や itch.io、README に貼るデモ GIF の作成手順。

SnapTack は「押す → 選ぶ → 残る」の一連が動いて初めて価値が伝わるツールで、
静止画では「ただのスクリーンショットツール」に見えてしまう。GIF は宣伝素材の中で
最も効果が大きいので、ここに手間をかける価値がある。

## 前提: 静止画で失敗しているポイント

`docs/images/screenshot.png` は付箋の枠線が細く、背景の VS Code と同系色のため、
**どこが付箋なのか一目で分からない**。GIF でも同じ失敗をしやすいので、下記の
「見せ方」を必ず守る。

## 1. 何を録るか (構成)

**全体で 6〜8 秒**を目安にする。X のタイムラインは数秒で流れるので、長いと
最後まで見られない (X 自身も短めを推奨している)。ファイルサイズも長さに比例するため、
短いほど 5MB に収めやすい。

ループ再生されるので、最後のフレームから最初に戻っても違和感が無いようにする。

| # | 秒数 | 内容 | 意図 |
|---|---|---|---|
| 1 | 0.5s | 何か作業中の画面を映す (Web ページや資料) | 「日常の場面」を示す |
| 2 | 0.5s | `Ctrl+Shift+Z` を押す → 画面が暗くフリーズ | 変化が起きたと分かる |
| 3 | 1.5s | ドラッグして範囲を選択 | **ゆっくり**動かす |
| 4 | 0.5s | 離す → その場に付箋が残る | ここが山場 |
| 5 | 1.5s | 付箋を別の場所へドラッグして移動 | 「浮いている」ことを示す |
| 6 | 1.5s | 元のウィンドウをスクロール / 別ウィンドウを前面に出す | **付箋だけが残る = 最前面**が伝わる |

**6 が最重要。** 背後のウィンドウが変化しても付箋が残る絵があって初めて
「常に最前面に貼りつく」という価値が伝わる。ここが無いと、ただの範囲キャプチャに見える。

余裕があれば末尾に「ホイールで不透明度変更」か「ダブルクリックで畳む」を足してもよいが、
**詰め込みすぎない**。1 つの GIF で伝えるのは 1 つの価値に絞る。

## 2. 録る前の準備 (画質を左右する)

- **切り取る対象は「文字が読めるもの」にする。** 無地の背景から切り取ると
  何をキャプチャしたのか分からない。文章・表・グラフなど、見て分かるものを選ぶ
- **付箋が背景に埋もれないようにする。** 明るい背景 (白いページ) の上に置くと
  黄色い枠が目立つ。暗いエディタの上だと同系色で埋もれる
- **デスクトップを片付ける。** 通知、個人情報、無関係なタブは事前に閉じる。
  **GIF は消せない**ので、映り込みには公開前に必ず目を通す
- **録画範囲は 1 ウィンドウ程度に絞る。** 4K 全画面を録ると縮小されて文字が潰れる。
  1280x720 前後に収まる範囲を録る
- **マウスカーソルを表示する設定**にする (どこを操作したか分からなくなるため)
- ドラッグは**普段の 1.5 倍ゆっくり**。実速度だと GIF では何が起きたか読み取れない

## 3. ツール別の設定値

### ScreenToGif (おすすめ)

録画からフレーム編集、GIF 出力までこれ 1 つで完結する。フレーム単位で
不要部分を削れるので、撮り直しが減る。

```powershell
winget install NickeManarin.ScreenToGif
```

| 設定 | 値 | 理由 |
|---|---|---|
| フレームレート | **15〜20 fps** | 30fps はファイルサイズが倍増する割に見た目が変わらない |
| 出力サイズ | **幅 800px 以下** | X のタイムライン表示で十分。それ以上は無駄 |
| エンコーダ | **FFmpeg** (無ければ 2.0 Gifski) | 色数の扱いが良い |
| 色数 | 256 | GIF の上限 |
| ループ | 無限 | |

出力後、**先頭と末尾の余分なフレームを削る**。起動直後の手探りの動きが残ると
素人っぽく見える。

### Xbox Game Bar (`Win+G`) — SnapTack のデモには使えない

追加インストール無しで録れるが、**このツールのデモには使えない**。

Game Bar はアプリ/ゲームのウィンドウ単位でしか録画できず、**デスクトップと
エクスプローラーは録画対象外**。試すと「ゲーム機能はデスクトップまたは
エクスプローラーでは利用できません」と表示される。これは設定で変えられない
設計上の制限。

SnapTack はオーバーレイを画面全体に出し、付箋をデスクトップ上に置くツールなので、
まさにこの制限に当たる。**ScreenToGif を使うこと。**

(参考) 別ツールで録った mp4 から GIF にする場合の変換 (要 ffmpeg):

```bash
ffmpeg -i demo.mp4 -vf "fps=15,scale=800:-1:flags=lanczos,split[a][b];[a]palettegen[p];[b][p]paletteuse" -loop 0 demo.gif
```

パレットを生成してから適用する 2 パス方式。単純変換より画質が明確に良くなる。

## 4. 仕上げの確認

- **ファイルサイズ**: **5MB 以下**を目標にする。
  X の上限は Web から 15MB だが、**モバイルアプリからは 5MB** と別枠。
  5MB 以下にしておけばどちらからでも投稿できる
  (なお X は GIF を受け取ると無音の mp4 に変換して再生する。
  そのため多少長くても再生自体は問題ないが、投稿時の上限は GIF のサイズで判定される)
- **1 周見て意味が伝わるか**: 音も説明も無い状態で理解できるか
- **映り込み**: 通知・個人情報・無関係なウィンドウが無いか

## 5. 置き場所

- `docs/images/demo.gif` に置く
- README の先頭付近に貼る (`README.md` と `docs/english-readme.md` の両方)
- itch.io は GIF をスクリーンショット欄にアップロードできる。**1 枚目に置く**
- X は投稿に直接添付する

サイズが大きい場合、リポジトリに入れるのを避けて Releases や外部に置く手もあるが、
5MB 以下なら気にせずコミットしてよい。
